Удаление (деинсталляция) комплекса ПО


Удаление необходимо производить в обратном порядке установки.

1. Отключить шелл на всех пользователях (где ранее был включен) всех клиентских машин (если используется клиентская часть с шеллом).
2. Отключить автологон на клиентских машинах (при необходимости).
3. Для клиентских частей с шеллом: на клиентских машинах запустить пункт "Удаление программы" в меню "ПУСК->Программы" Windows.
3. Для клиентских частей без шелла: через программу оператора для всех клиентских машин запустить "Удаление клиентской части".
4. Закрыть программу оператора и из-под администратора машины запустить пункт "Удаление программы" в меню "ПУСК->Программы" Windows.
5. Аналогично удалить и программу администратора на администраторской машине.
6. Удалить сервер Runpad Pro таким же способом.

Следует заметить, что все настройки сохранятся даже после удаления.
Это касается также и базы данных Runpad Pro (информация только для MSSQL Server).
При желании вы можете удалить и SQL-сервер (через "Панель управления->Установка и удаление программ"), однако сам файл БД Runpad Pro останется (информация только для MSSQL Server), чтобы впоследствии можно было его снова подключить к базе. Если пожелаете его удалить, то сделать это нужно будет вручную из папки SQL-сервера (обычно это "\Microsoft SQL Server\MSSQL\Data\RunpadPro.*") (информация только для MSSQL Server)
Для MySQL: останутся файлы БД после его удаления или нет зависит от конкретной версии сервера.
