
rshell_asv2 - утилита для интеграции RunpadPro и Astalavista2.
Интеграция включает:
1) выдача информации про сеанс пользователя на рабочий стол шелла;
2) выдача информации в системный трей;
3) управление закладками шелла в зависимости от оплаченного посетителем тарифа;
4) автоматический VIP-логин клиента в шелле.

Замечания:
* Утилита работает только совместно с RunpadPro (RunpadShell не поддерживается).
* Нужно использовать RunpadPro версии 1.35 и выше.
* Утилита не будет работать совместно с программой InfoASV2.exe.
* Точно также утилита InfoASV2.exe не будет работать совместно с rshell_asv2.
* Утилита будет работать только если ранее был куплен АстаШелл или соотв. функциональность в Astalavista2
* Для добавления программы в автозагрузку текущего пользователя (HKEY_CURRENT_USER) используйте: 
  rshell_asv2.exe /autorunCU
* Для добавления программы в автозагрузку всех пользователей (HKEY_LOCAL_MACHINE) используйте: 
  rshell_asv2.exe /autorunLM
  (будут нужны права администратора системы!)
* Для удаления программы из автозагрузки используйте:
  rshell_asv2.exe /uninstall
  (для удаления из HKEY_LOCAL_MACHINE будут нужны права администратора системы!)
* Описание настроек и примеры настроек находятся в файле rshell_asv2.ini

Рекомендации:
* Рекомендуется добавить rshell_asv2.exe в список разрешенных иконок трея в настройках Runpad.
* Можно также добавить rshell_asv2.exe в опцию "Восстановление клиента" в настройках Runpad.
* Убедитесь, что включена опция "Игнорировать пароль при VIP-логине через API шелла" в
  настройках шелла на вкладке "VIP-клиенты".
* Для большей безопасности добавляйте утилиту в автозагрузку HKEY_LOCAL_MACHINE,
  а работайте под ограниченным пользователем.
* Более удобный и самый безопасный способ - добавить программу в автозагрузку 
  на вкладке настроек шелла - "Обслуживание: Автозагрузка".
* Для большей безопасности файл настроек rshell_asv2.ini храните в папке Windows 
  (там же можно хранить и rshell_asv2.exe).
* Для быстрого обновления/добавления файла rshell_asv2.ini (rshell_asv2.exe) на все машины 
  при его хранении в папке Windows используйте средства автоматического обновления RunpadPro:
  1) в программе администратора в папке "C:\Program Files\Runpad Pro Admin\update\client\"
     создайте папку "WINDIR" и в нее перепишите ваш rshell_asv2.ini (rshell_asv2.exe);
  2) запустите программу начальной конфигурации БД RunpadPro - rsdbconf.exe, чтобы занести
     этот файл (файлы) в SQL-базу;
  3) в программе оператора на выбранных машинах выполните обновление клиентского шелла,
     после чего ваш файл rshell_asv2.ini (rshell_asv2.exe) попадет в папку \Windows 
     каждого клиентского компьютера.
  Также можно использовать и удаленный файловый менеджер шелла для этих целей.
